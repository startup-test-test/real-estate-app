# SEC-018: 借入額と購入価格の論理検証

## 概要
不動産投資シミュレーターにおいて、借入額と購入価格の論理的整合性をチェックし、非現実的な資金計画を防止する機能を実装しました。

## 実装内容

### 1. 検証ロジックの強化
`validateLoanToPurchaseRatio`関数を拡張し、以下の検証を実装：

1. **基本的な妥当性チェック**
   - 購入価格が0以下の場合はエラー
   - 借入額が負の値の場合はエラー

2. **総取得価格の考慮**
   - 総取得価格 = 購入価格 + 諸経費 + 改装費
   - 借入額が総取得価格を超える場合は絶対にNG

3. **LTV（Loan to Value）制限**
   - 借入額が購入価格を超える場合はエラー（100%超）
   - LTVが90%を超える場合は警告（一般的な融資基準）

4. **自己資金比率のチェック**
   - 自己資金がマイナスになる場合はエラー
   - 自己資金比率が10%未満の場合は警告

### 2. 実装コード
```typescript
export function validateLoanToPurchaseRatio(
  loanAmount: number | null,
  purchasePrice: number | null,
  otherCosts: number = 0,
  renovationCost: number = 0
): { isValid: boolean; message?: string; ltv?: number }
```

### 3. エラーメッセージの改善
- 具体的な金額を表示
- 推奨される基準を明記
- ユーザーが理解しやすい説明

## 使用例

### 正常なケース
```typescript
// 購入価格7000万円、借入額5000万円（LTV 71.4%）
validateLoanToPurchaseRatio(5000, 7000, 300, 200)
// => { isValid: true, ltv: 71.4 }
```

### エラーケース
```typescript
// 借入額が購入価格を超える
validateLoanToPurchaseRatio(8000, 7000)
// => { isValid: false, message: "借入額（8,000万円）が購入価格（7,000万円）を超えています..." }

// 自己資金不足
validateLoanToPurchaseRatio(6800, 7000, 300, 200)
// => { isValid: false, message: "自己資金比率が9.3%です。最低でも10%以上..." }
```

## セキュリティ効果

1. **不正な資金計画の防止**
   - 非現実的な借入額の入力を制限
   - 金融機関の融資基準に沿った検証

2. **計算エラーの防止**
   - 論理的に矛盾した値での計算を回避
   - システムの信頼性向上

3. **ユーザー保護**
   - 危険な資金計画への警告
   - 適切な自己資金比率の確保を促進

## 影響範囲
- シミュレーター入力画面
- 購入価格、借入額、諸経費、改装費の入力フィールド
