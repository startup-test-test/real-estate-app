# デザインガイドライン

## 概要

大家DXのUIデザインルール。新規ページ・コンポーネント作成時はこのガイドラインに従うこと。

---

## 技術スタック

| 技術 | バージョン | 用途 |
|------|-----------|------|
| Tailwind CSS | v4 | ユーティリティCSS |
| shadcn/ui | new-york スタイル | UIコンポーネント基盤 |
| Lucide React | - | アイコン（`components/ui/icons.tsx` で一元管理） |
| class-variance-authority (cva) | - | コンポーネントの variant 管理 |

### shadcn/ui コンポーネント追加方法

```bash
npx shadcn@latest add <component-name>
```

追加されたコンポーネントは `components/ui/` に配置される。

| `primary-400` | `#38bdf8` | - |
| `primary-500` | `#0ea5e9` | フォーカスリング |
| `primary-600` | `#0284c7` | **メインボタン、リンク** |
| `primary-700` | `#0369a1` | ホバー時ボタン |
| `primary-800` | `#075985` | - |

### グレー（Tailwind デフォルト）

| 用途 | クラス |
|------|--------|
| ページ背景 | `bg-gray-50` |
| カード背景 | `bg-white` |
| 本文テキスト | `text-gray-900` |
| 補助テキスト | `text-gray-700` |
| キャプション | `text-gray-500` |
| ボーダー | `border-gray-200` |
| 区切り線 | `border-gray-300` |

### shadcn/ui CSS変数

shadcn/ui コンポーネント内部で使用。`globals.css` の `:root` で定義。

| 変数 | 対応色 | 説明 |
|------|--------|------|
| `--primary` | primary-600 相当 | shadcn コンポーネントのアクセント |
| `--background` | gray-50 相当 | ページ背景 |
| `--foreground` | gray-900 相当 | テキスト |
| `--border` | gray-200 相当 | ボーダー |
| `--ring` | primary-600 相当 | フォーカスリング |
| `--destructive` | red 系 | エラー・削除 |

### 使い分けルール

- **既存のカスタムコンポーネント** → `primary-600`、`gray-900` 等の Tailwind クラスを使用
- **shadcn/ui コンポーネント** → CSS変数（`bg-primary`、`text-foreground` 等）を自動使用
- 両者は同じ色に設定されているため、見た目は統一される

## コンポーネント規約

### ボタン

shadcn/ui の `Button` コンポーネント（`components/ui/button.tsx`）を使用する。

```tsx
import { Button } from '@/components/ui/button'

// プライマリ
<Button>保存する</Button>

// セカンダリ
<Button variant="secondary">キャンセル</Button>

// アウトライン
<Button variant="outline">詳細を見る</Button>

// ゴースト（ナビゲーション等）
<Button variant="ghost">メニュー</Button>

// リンク風
<Button variant="link">もっと見る</Button>
```

**旧スタイル（`.btn-primary`、`.btn-secondary`）は新規コードでは使わない。**

### カード

```tsx
<div className="bg-white rounded-xl border border-gray-200 shadow-soft p-6">
  {/* content */}
</div>
```

shadcn/ui の `Card` を追加した場合はそちらを優先する。

### フォーム入力

shadcn/ui の `Input`、`Select` 等を追加後に使用すること。追加前は既存の `.input` クラスを使用。

### アイコン

`components/ui/icons.tsx` に定義済みのものを優先使用する。未定義のアイコンは Lucide React から直接 import してもよい。

```tsx
// 推奨：一元管理されたアイコン
import { Icons } from '@/components/ui/icons'
<Icons.check className="h-4 w-4" />

// OK：直接 import
import { Calculator } from 'lucide-react'
<Calculator className="h-4 w-4" />
```

## ダークモード

現時点では**未対応**。`globals.css` に `.dark` の CSS変数は定義済みだが、切り替え機能は実装していない。将来的に対応する場合は shadcn/ui の `dark:` variant をそのまま活用できる。

## ファイル構成

```
components/
  ui/               ← shadcn/ui コンポーネント（自動生成）
    button.tsx
    icons.tsx
    (今後追加: input.tsx, card.tsx, select.tsx, tabs.tsx ...)
  tools/             ← ツール固有コンポーネント
  landing/           ← ランディングページ固有
  shared/            ← 共通コンポーネント
  calculators/       ← 計算ツール共通コンポーネント
```

### 命名規則

- コンポーネントファイル: `PascalCase.tsx`（例: `CompanyProfileCompact.tsx`）
- ユーティリティ: `camelCase.ts`（例: `utils.ts`）
- shadcn/ui: `kebab-case.tsx`（例: `button.tsx`）— shadcn CLI が生成する命名に従う
