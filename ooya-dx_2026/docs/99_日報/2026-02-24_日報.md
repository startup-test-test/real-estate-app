### 2. ExperienceBoxコンポーネント新規作成

記事ごとに固有の体験データを表示するための新コンポーネントを作成。

| 項目 | 内容 |
|------|------|
| ファイル | `components/media/ExperienceBox.tsx` |
| 用途 | 枠（デザイン）だけ提供、中身はMDXで自由に記述 |
| デザイン | SummaryBoxと同じ青系（primary）デザイン |

#### ExperienceTable（既存）との使い分け

| コンポーネント | 用途 |
|--------------|------|
| ExperienceTable | 全記事共通の購入7件・売却1件の一覧表（データはハードコード） |
| **ExperienceBox** | 記事ごとの固有の体験（ローン内訳、交渉の経緯など。MDXで自由記述） |

#### housing-loan.mdxでの使い方

```
<ExperienceBox title="結論と著者の実体験｜オーバーローンの全内訳">
  ① 結論と実体験（箇条書き）
  ② 著者が住宅ローンに諸費用を組み込んだ内訳（テーブル）
</ExperienceBox>
```

### 4. 体験型タイトルの方針決定

| 決定事項 | 内容 |
|---------|------|
| タイトル冒頭のラベル | 全記事【実体験】ではなく、記事ごとに変える（【実録】【実額公開】【売買8回】等） |
| housing-loan.mdx | 【実録】を採用 |
| legal-limit.mdxの「違法」KW | 法的リスクを避けて見送り。現状の「請求された金額に疑問を感じたら」を維持 |
| negotiation.mdxの「言い方・例文」 | 対応方針は次回検討 |
| テンプレート方針 | 全記事が実体験ベースの前提でテンプレートを統一する |

#### テンプレート構成（決定）

```
① QuestionBox（読者の疑問）
② ExperienceBox（結論 + この記事に関連する実体験データ）
③ 本文（H2〜）
④ FAQ
⑤ まとめ
```

- SummaryBoxはExperienceBoxに統合して廃止する方針
- ExperienceTable（共通データ）はguide.mdxなど必要な記事のみ使用
