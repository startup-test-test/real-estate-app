## 3. 検討した選択肢

### 3.1 案A：両ドメイン並存（却下）

```
startup-marketing.co.jp → 法人サイトとして維持
ooya.tech → サービスサイト
相互リンクでSEO効果を分散
```

**メリット:**
- 政府リンクを自然な文脈で維持
- リスクが低い

**デメリット:**
- ドメインパワーが分散
- 2サイト運用のコスト
- ブランドが分散

### 3.2 案B：リダイレクト統合（採用）

```
startup-marketing.co.jp → ooya.tech/company/ へ301リダイレクト
全ての資産をooya.techに集約
```

**メリット:**
- ドメインパワーの集中
- ブランドの統一
- 運用コスト削減
- 政府リンクの価値をooya.techに移転

**デメリット:**
- 政府リンク削除リスク（後述で検証）

## 4. リスク分析

### 4.1 当初の懸念

1. **トピック関連性の問題**
   - 政府リンクは「IT企業」「SDGs企業」としての評価
   - 不動産サイトにリダイレクトすると関連性が薄い？

2. **政府リンク削除リスク**
   - 厚労省等がリンク先を確認した際に削除される可能性

3. **アンカーテキストの不一致**
   - 「株式会社startupmarketing」「ホームページ制作会社」等のアンカー
   - 不動産キーワードとは無関係

### 4.2 リスクの再評価

**政府リンク削除リスクは極めて低い**

理由:
1. リダイレクト先（ooya.tech/company/）に同じ法人情報が表示される
2. 政府機関の視点では「同じ会社の情報ページ」に遷移するだけ
3. 会社が変わったわけではなく、Webサイト構成が変わっただけ
4. 政府機関のリンク先チェックは頻繁に行われない

**削除される条件（該当しない）:**
- ❌ リンク先が無関係なサイトになった → 会社情報ページなので該当しない
- ❌ 会社が廃業した → 事業継続中
- ❌ 詐欺サイトにリダイレクトされた → 正当な事業サイト

### 4.3 現状の整理

**政府・公的機関リンクは法人登録が必要（ドメイン種類は問わない）**

- がん対策推進企業アクション → 法人登録が必要
- ポジティブ・オフ → 法人登録が必要
- SDGsパートナー → 法人登録が必要

**注意**: `.co.jp` でも `.tech` でもドメイン種類は関係ない。株式会社StartupMarketingとして登録する際に、会社URLとして `ooya.tech/company/` を申請すれば、ooya.techに直接リンクをもらうことも可能。

**現状**: 過去に startup-marketing.co.jp で各プログラムに登録済みのため、既存リンクは startup-marketing.co.jp に向いている。

**選択肢:**
1. 各政府プログラムに会社URL変更を申請 → ooya.tech に直接リンク（手間がかかる）
2. 301リダイレクト → 既存リンクの価値を移転（効率的）

## 5. 最終決定

### 5.1 結論

**startup-marketing.co.jp → ooya.tech へ301リダイレクトを実施**

### 5.2 決定理由

1. 政府系リンク（DA 89-91）は既に startup-marketing.co.jp に向いている貴重な資産
2. 各プログラムへのURL変更申請より、301リダイレクトの方が効率的
3. リダイレクト先に同じ法人情報を表示すれば、削除リスクは極めて低い
4. 事業を不動産テックに特化する方針と一致
5. 1つのドメインに集約することでブランド力・SEO効果が最大化

### 5.3 期待効果

| 項目 | 現状 | リダイレクト後（予測） |
|------|------|----------------------|
| ooya.tech DA | 5 | 8-12 |
| 政府リンクの恩恵 | なし | あり |
| ブランド認知 | 分散 | 統一 |

### 6.4 WordPress .htaccess 設定例

```apache
# 301リダイレクト設定
<IfModule mod_rewrite.c>
RewriteEngine On

# 個別ページのリダイレクト
RewriteRule ^contact/?$ https://ooya.tech/contact/ [R=301,L]
RewriteRule ^service/?$ https://ooya.tech/services/ [R=301,L]
RewriteRule ^news/(.*)$ https://ooya.tech/media/ [R=301,L]

# その他全てをcompanyへ
RewriteRule ^(.*)$ https://ooya.tech/company/ [R=301,L]
</IfModule>
```

### 6.5 ブログリダイレクトの判断根拠

**410 Gone ではなく 301リダイレクトを選択した理由:**

1. `/blog/` への被リンクが存在（DA 40-50程度のサイトから）
2. 被リンクのリンクジュースを ooya.tech に移転
3. ユーザーが404ページに遭遇しない
4. トラフィックが少ないため、UX影響は軽微

## 8. 補足：TLDについて

**.co.jp → .tech のリダイレクトに問題はない**

- GoogleはTLD（ドメイン末尾）でSEO評価を変えない
- .co.jp も .tech も .com も同等に扱われる
- 301リダイレクトのリンクジュース移転はTLDに関係なく機能
- .co.jp の法人信頼性は、リダイレクト先に引き継がれる
