# 見出し構造ルール（h2/h3）

シミュレーターページの見出し（h2/h3）を決める際のルールです。
SEO観点と読みやすさを考慮した基本原則をまとめています。

---

## 1. 基本原則

| 見出し | 役割 | 数の目安 | 文字数目安 |
|--------|------|----------|------------|
| **h1** | ページタイトル（1ページ1つ） | 1つ | 25〜35文字 |
| **h2** | 大テーマ・章 | 3〜7個 | 10〜25文字 |
| **h3** | h2の詳細・節 | 各h2の下に0〜4個 | 10〜20文字 |

### 余白ルール（spacing）

| 見出し | 上の余白 | 下の余白 | スクロール調整 | Tailwind |
|--------|----------|----------|----------------|----------|
| **h2** | 48px | 16px | 96px | `mt-12 mb-4 scroll-mt-24` |
| **h3** | 32px | 12px | 96px | `mt-8 mb-3 scroll-mt-24` |

※ `scroll-mt-24` は固定ヘッダーの高さ分のスクロールマージン。アンカーリンクで飛んだ際にヘッダーと被らないようにする。

### 構造のイメージ

```
h1: ページタイトル（1つのみ）
│
├── h2: 大テーマ1
│   ├── h3: 詳細1-1
│   ├── h3: 詳細1-2
│   └── h3: 詳細1-3
│
├── h2: 大テーマ2
│   ├── h3: 詳細2-1
│   └── h3: 詳細2-2
│
├── h2: 大テーマ3（h3なしもOK）
│
└── h2: 大テーマ4
    └── h3: 詳細4-1
```

---

## 2. h2の決め方

### 基本ルール

**h2 = ユーザーが検索しそうなキーワード・質問**

h2はページの「章」にあたります。ユーザーが検索エンジンで入力しそうなキーワードを含めることで、SEO効果が期待できます。

### 良いh2の例

| パターン | 例 | 検索意図 |
|----------|-----|----------|
| 「〇〇とは」 | 仲介手数料とは | 定義を知りたい |
| 「〇〇の計算方法」 | 贈与税の計算方法 | やり方を知りたい |
| 「〇〇がかからないケース」 | 仲介手数料がかからないケース | 例外を知りたい |
| 「〇〇の注意点」 | 相続税の注意点 | 失敗を避けたい |
| 「〇〇年の法改正」 | 2024年の法改正について | 最新情報を知りたい |
| 「〇〇と△△の違い」 | 売買と賃貸の違い | 比較したい |
| 「〇〇の特例・控除」 | 主な特例・控除制度 | 節税したい |

### 避けるべきh2

| NG例 | 理由 |
|------|------|
| 「詳細」「その他」「補足」 | 検索されない、内容が不明 |
| 「ポイント」「まとめ」 | 具体性がない |
| 内容が1〜2行しかない見出し | h3に降格すべき |

---

## 3. h3の決め方

### 基本ルール

**h3 = h2を補足する詳細情報**

h3はh2の「節」にあたります。h2のトピックを具体的に説明するために使います。

### h3の使い方

```
h2: 仲介手数料の計算方法
├── h3: 速算式（3%+6万円）
├── h3: 正式な料率（段階制）
└── h3: 具体的な計算例

h2: 支払いと仕組み
├── h3: 支払いのタイミング
└── h3: 両手仲介と片手仲介
```

### h3を使う基準

- h2だけでは説明しきれない詳細がある
- 2つ以上の小トピックに分けられる
- 各h3に十分な内容（3行以上）がある

---

## 4. 判断フローチャート

新しい見出しを追加するとき、以下のフローで判断します。

```
その見出しは検索キーワードになりそう？
│
├── YES → h2にする
│
└── NO → 上位トピック（h2）の詳細説明？
         │
         ├── YES → h3にする
         │
         └── NO → h2に格上げ or 見出しにしない
```

### 具体例

| 見出し候補 | 検索される？ | 判断 |
|------------|--------------|------|
| 「仲介手数料とは」 | YES（定義検索） | → h2 |
| 「速算式」 | NO（単体では検索されにくい） | → h3 |
| 「2024年法改正」 | YES（最新情報検索） | → h2 |
| 「支払いのタイミング」 | 微妙 | → h3（「支払い」h2の下） |
| 「仲介手数料 かからない」 | YES（例外検索） | → h2 |

---

## 5. NGパターンと対処法

| パターン | 問題点 | 対処法 |
|----------|--------|--------|
| h2が1つでh3が10個 | 構造が浅い、読みにくい | h3をグループ化してh2を増やす |
| h3が孤立（h2の下に1つだけ） | 不自然な構造 | h2に統合 or h3を追加 |
| h2が10個以上 | 散漫で読みにくい | 関連するものをh3に統合 |
| h2とh3で内容が重複 | 冗長 | どちらかに統一 |
| h3の下にh3（階層違反） | HTML的にNG | h2/h3の2階層に整理 |

## 6. シミュレーターページの推奨構造

### 標準テンプレート

```
h1: [ツール名]シミュレーター

h2: [ツール対象]とは
├── h3: 計算方法
├── h3: 税率・料率
└── h3: 具体的な計算例

h2: [特例・控除]について
├── h3: 特例1
└── h3: 特例2

h2: [重要な注意点・法改正]

h2: [関連する知識]
├── h3: 関連トピック1
└── h3: 関連トピック2
```

### 仲介手数料ページの例

```
h1: 不動産売買の仲介手数料 計算シミュレーション｜早見表付き

h2: 1. 仲介手数料とは
├── h3: 1-1. 計算方法（速算式）
├── h3: 1-2. 正式な料率
└── h3: 1-3. 具体的な計算例

h2: 2. 2024年法改正について

h2: 3. 支払いと仕組み
├── h3: 3-1. 支払いのタイミング
└── h3: 3-2. 両手仲介と片手仲介

h2: 4. 仲介手数料がかからないケース

h2: 5. 売買と賃貸の違い
```

---

## 7. チェックリスト

新しいページを作成したら、以下を確認してください。

- [ ] h1は1つのみか
- [ ] h2は3〜7個の範囲か
- [ ] h2に検索キーワードが含まれているか
- [ ] h3が孤立していないか（h2の下に1つだけは避ける）
- [ ] h2/h3の内容が重複していないか
- [ ] 各見出しに十分な本文があるか（h2: 3行以上、h3: 2行以上）
