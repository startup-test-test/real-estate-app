不動産登記にかかる登録免許税シミュレーター開発に向けた包括的調査報告書
第1章 調査の目的と背景
1.1 報告書の目的
本報告書は、不動産取引における「登録免許税」の試算シミュレーターを新規開発・実装するために必要な、税制、計算ロジック、法的要件、および市場環境を網羅的に調査・分析したものである。特に、複雑化する軽減税率（特例措置）の適用条件や、2026年（令和8年）以降を見据えた法改正の動向、さらには競合他社ツールとの差別化要因となるユーザー体験（UX）の設計指針を提供することを主眼としている。
登録免許税は、不動産購入時の諸費用の中でも大きな割合を占めるが、その算出根拠となる「課税標準額」の考え方が一般消費者にとって難解であり、かつ「軽減措置」の適用可否が物件のスペックや取得時期によって細かく変動するため、正確な見積もりを出すことが困難な領域である。本報告書では、エンジニアリングチームが実装可能なレベルまでロジックを分解し、かつプロダクトマネージャーが市場ニーズを把握できる深度で情報を提供する。
1.2 調査の範囲
本調査は以下の領域をカバーする。
税制の基本構造: 納税義務者、課税のタイミング、不動産登記の種類ごとの基本税率。
計算アルゴリズム: 固定資産税評価額と法務局認定価格の使い分け、端数処理（切り捨て）の厳密なルール。
軽減措置（特例）の全容: 住宅用家屋、認定長期優良住宅、認定低炭素住宅、買取再販住宅など、あらゆるパターンの適用要件と税率。
法改正のタイムライン: 土地の売買にかかる特例の延長（令和8年・令和11年問題）、住宅特例の期限（令和9年）の整理。
周辺コスト: 司法書士報酬の市場相場と計算モデル。
実装要件: シミュレーターに必要な入力フィールド、バリデーションルール、出力項目。
第2章 登録免許税の法的性質と基本構造
シミュレーターの基礎となるのは、登録免許税法および関連する租税特別措置法の理解である。ここでは、システム設計の前提となる法的枠組みを定義する。
2.1 登録免許税の定義と課税根拠
登録免許税（Registration and License Tax）は、登記、登録、特許、免許、許可、認可、認定、指定および技能証明について課される国税である。不動産登記においては、所有権の保存、移転、抵当権の設定などの際に、登記を受ける者が国に対して納付する義務を負う。
課税のタイミング: 登記の申請時である。実務上は、不動産の決済日（引き渡し日）に司法書士が法務局へ申請書を提出する直前に納付される 1。
納税義務者: 法的には、登記権利者（買主など）と登記義務者（売主など）が連帯して納付する義務を負うが、不動産取引の慣習上、所有権移転登記や抵当権設定登記の費用は**「買主（権利取得者）」**が全額負担することが一般的である 2。
納付方法: 原則として現金納付（金融機関等での支払い）を行い、領収証書を登記申請書に貼り付ける。ただし、税額が3万円以下の場合は収入印紙での納付も認められている。また、オンライン申請の場合は電子納付（Pay-easy等）が可能である 3。

第3章 課税標準（Tax Base）の算出ロジック詳細
登録免許税の計算式はシンプルに 「課税標準額 × 税率」 であるが、この「課税標準額」の特定こそが最大の難関であり、シミュレーターの精度を左右する。
3.1 課税標準額の原則：固定資産税評価額
原則として、登録免許税の課税標準額は、市町村の固定資産課税台帳に登録されている**「固定資産税評価額」**である。これは実際の「売買価格（市場価格）」とは明確に異なる数値である 2。
データソース: 毎年4〜5月に所有者に送付される「固定資産税納税通知書」の添付書類（課税明細書）に記載されている「価格」または「評価額」の欄を参照する。
注意点: 「課税標準額」という似た用語が固定資産税の計算に使われるが、登録免許税の計算においては「本則の価格（評価額）」そのものを使用する（※土地の固定資産税計算では特例で課税標準が下がっている場合があるが、登録免許税は評価額ベースである点に注意が必要） 7。
シミュレーターにおける課題と解決策
ユーザーは購入検討段階では「評価額」を知らないことが多い。物件価格（売買代金）しか入力できないユーザーに対し、内部ロジックで評価額を推定する必要がある。
推定ロジックの実装:
土地: 市場価格（売買価格）の 約70% と推定 1。
中古建物: 市場価格の 約50〜60% と推定（ただし築年数により大きく変動するため、免責事項が必須） 1。
UI要件: 「固定資産税評価額がわかる場合は入力してください。わからない場合は売買価格から概算します」というトグルスイッチまたは入力欄の切り替え機能を設けるべきである。
3.2 例外：新築建物の課税標準（法務局認定価格）
新築建物（完成直後）には、まだ固定資産税評価額が設定されていない。この場合、国（法務局）が定めた**「新築建物課税標準価格認定基準表」**に基づき、みなし評価額を算出する 1。
計算式

$$\text{課税標準額} = \text{床面積}(m^2) \times \text{認定単価}(\text{円}/m^2)$$
認定単価の変動要因:
管轄法務局（地域）: 東京法務局と大阪法務局では単価が異なる。
構造: 木造、鉄骨造、鉄筋コンクリート造（RC）などで単価が異なる。
種類: 居宅、共同住宅、店舗などで異なる。
データ例（令和6年度 東京法務局） 8:
木造・居宅: 107,000円/$m^2$
鉄筋コンクリート造・居宅: 130,000円/$m^2$
鉄骨造・居宅（軽量鉄骨）: 121,000円/$m^2$
実装要件: シミュレーターは、都道府県と構造（木造/RC/鉄骨）を選択させることで、バックエンドのデータベースから適切な単価を呼び出し、ユーザーが入力した床面積と掛け合わせる機能を実装する必要がある。
3.3 端数処理（Rounding Rules）の厳密な仕様
登録免許税法および国税通則法に基づく端数処理は、プログラム上で正確に再現しなければならない。1円のズレも許されない。
① 課税標準額の端数処理（切り捨て）
課税標準額（評価額の合計）に対し、1,000円未満を切り捨てる 3。
ロジック: Math.floor(評価額 / 1000) * 1000
例外: 評価額が1,000円未満の場合は、1,000円とする（切り捨てて0円にはしない）。
② 税額の端数処理（切り捨て）
算出された税額（課税標準額 × 税率）に対し、100円未満を切り捨てる 3。
ロジック: Math.floor((課税標準額 * 税率) / 100) * 100
最低税額（足切り）: 計算結果が1,000円未満になる場合は、一律 1,000円 とする（免税でない限り）。
住宅ローン（抵当権設定）の特殊ルール
抵当権設定登記の課税標準は「債権金額（ローン借入額）」である 2。
ロジック: Math.floor(借入額 / 1000) * 1000 × 税率
同様に、税額の100円未満切り捨て、最低1,000円ルールが適用される。

4.2 軽減措置（特例）の適用条件ロジック
シミュレーターでは、以下の条件をチェックボックスや入力項目で確認し、適用可能な税率を自動判定する機能が求められる 11。
共通要件（住宅用家屋証明書の取得条件）
自己居住用であること: 投資用、セカンドハウス（一部例外あり）は対象外。「自分が住むための家ですか？」という質問項目が必須 17。
床面積要件:
原則: 登記簿上の床面積が 50$m^2$以上 であること。
緩和措置: 2023年4月1日以降に建築確認を受けた家屋等の場合、合計所得金額が1,000万円以下であれば 40$m^2$以上 でも適用可能（令和8年末までの時限措置等の確認が必要） 15。
登記のタイミング: 新築または取得から 1年以内 に登記を行うこと 11。
耐震基準（中古住宅の場合）:
昭和57年（1982年）1月1日以降に建築された家屋（新耐震基準）。
それ以前の場合でも、耐震基準適合証明書や既存住宅売買瑕疵保険への加入があれば可 14。
インサイト: 以前あった「木造20年、非木造25年」という築年数制限は廃止されており、新耐震基準適合が絶対条件となっている点に注意 14。
個別特例の判定
長期優良住宅: 「長期優良住宅の認定を受けていますか？」 → Yesなら保存登記0.1%。
低炭素住宅: 「低炭素住宅の認定を受けていますか？」 → Yesなら保存登記0.1%。
買取再販: 「リフォーム済みの物件を不動産会社から購入しますか？（安心R住宅など）」 → Yesなら移転登記0.1%（通常の中古軽減0.3%より有利） 13。
第5章 2026年・2027年に向けた法改正と適用期限の動向
シミュレーターの寿命を左右するのが、法改正による「適用期限（サンセット条項）」の管理である。調査時点で判明している最新の期限情報は以下の通りであり、これらは今後の税制改正大綱により延長される可能性が高い。
5.1 土地の売買による所有権移転登記の特例（1.5%）
土地の移転登記の本則は2.0%だが、長らく特例により1.5%に据え置かれている。
現状の期限: 令和8年（2026年）3月31日 まで 4。
最新動向（令和7年度税制改正要望等）: 国土交通省等は、この期限をさらに3年間延長し、令和11年（2029年）3月31日 までとすることを要望・決定プロセスにある 11。
シミュレーター対応: デフォルト値を1.5%としつつ、注釈で「※本則は2.0%ですが、特例措置（令和8年/11年まで延長予定）を適用しています」と表示する設計が安全である。
5.2 住宅用家屋の軽減税率（0.3% / 0.15% / 0.1%）
建物の保存・移転・抵当権設定に関わる住宅用特例の期限。
現状の期限: 令和9年（2027年）3月31日 まで 11。
解説: 令和6年度税制改正により、従来の期限から3年延長されたばかりであるため、当面（2026年時点）はこの期限が有効である。シミュレーター上では2027年3月末までの登記日であれば軽減税率を適用して問題ない。
5.3 相続登記の免税措置
相続登記の促進（所有者不明土地問題の解消）のため、以下の免税措置が設けられている 23。
数次相続の免税: 相続人が登記をする前に死亡した場合、その死亡した相続人への移転登記は免税。
少額土地の免税: 土地の価額（評価額）が 100万円以下 の場合、相続登記の登録免許税は免税。
期限: これらも令和7年度改正プロセスにおいて、令和9年（2027年）3月31日 まで延長される見込みである 23。
シミュレーター対応: 入力された土地評価額が100万円以下かつ登記原因が「相続」の場合、自動的に税額を0円にするロジックの実装が推奨される。

const RATE_BLDG_PRES_GEN = 0.0015; // 0.15%
const RATE_BLDG_TRANS_USED = 0.003; // 0.3%
const RATE_MORTGAGE_GEN = 0.001; // 0.1% (軽減時)
const NEW_BUILD_UNIT_PRICE_WOOD = 107000; // 木造単価

function calculateTax(inputs) {
  let landTax = 0;
  let bldgTax = 0;
  let mortgageTax = 0;

  // --- 1. 土地の計算 ---
  let landBase = inputs.landAssessedValue;
  if (!landBase) {
    landBase = inputs.landMarketPrice * 0.7; // 推定ロジック
  }
  // 課税標準の端数処理（1000円未満切り捨て）
  landBase = Math.floor(landBase / 1000) * 1000;
  if (landBase < 1000) landBase = 1000;
  
  // 相続時の少額免税判定
  if (inputs.type === 'Inheritance' && landBase <= 1000000) {
    landTax = 0;
  } else {
    let landRate = (inputs.type === 'Sale')? RATE_LAND_TRANSFER_SALE : 0.02; // 本則
    if (inputs.type === 'Inheritance') landRate = 0.004;
    
    landTax = Math.floor((landBase * landRate) / 100) * 100;
    if (landTax < 1000) landTax = 1000;
  }

  // --- 2. 建物の計算 ---
  let bldgBase = 0;
  if (inputs.bldgType === 'New') {
    // 新築：認定価格計算
    let unitPrice = getUnitPrice(inputs.prefecture, inputs.structure);
    bldgBase = inputs.floorArea * unitPrice;
  } else {
    // 中古：評価額
    bldgBase = inputs.bldgAssessedValue |

| (inputs.bldgMarketPrice * 0.6);
  }
  
  // 建物の課税標準端数処理
  bldgBase = Math.floor(bldgBase / 1000) * 1000;
  if (bldgBase < 1000) bldgBase = 1000;

  // 税率判定（特例適用チェック）
  let bldgRate = 0.02; // デフォルト中古
  if (inputs.isResidential && inputs.floorArea >= 50) {
    if (inputs.bldgType === 'New') {
      bldgRate = inputs.isLongTerm? 0.001 : RATE_BLDG_PRES_GEN;
    } else {
      bldgRate = inputs.isResale? 0.001 : RATE_BLDG_TRANS_USED;
    }
  }
  
  bldgTax = Math.floor((bldgBase * bldgRate) / 100) * 100;
  if (bldgTax < 1000) bldgTax = 1000;

  // --- 3. 抵当権の計算 ---
  if (inputs.hasLoan) {
    let loanBase = Math.floor(inputs.loanAmount / 1000) * 1000;
    let mortgageRate = (inputs.isResidential && inputs.floorArea >= 50)? 0.001 : 0.004;
    
    mortgageTax = Math.floor((loanBase * mortgageRate) / 100) * 100;
    if (mortgageTax < 1000) mortgageTax = 1000;
  }

  return {
    totalTax: landTax + bldgTax + mortgageTax,
    details: { land: landTax, building: bldgTax, mortgage: mortgageTax }
  };
}

第9章 競合ツール分析と差別化要件
現在Web上に存在する登録免許税シミュレーターの多くは、以下の欠点を抱えている。
競合ツールのタイプ
特徴
欠点・ユーザーの不満点
大手ポータル系
簡易的。「物件価格」だけで計算する。
「評価額」ではなく「売買価格」に税率を掛けてしまい、数倍〜数十倍の高額な誤算を表示するケースが多い。
司法書士事務所系
正確。評価額入力を求める。
UIが古く、スマホ非対応。専門用語（課税標準等）の解説がなく、一般ユーザーが入力できない（評価額が手元にないため離脱する）。
金融機関系
ローン計算のついでに表示。
概算すぎて、「軽減税率」の詳細設定（長期優良など）が反映されない。

差別化のための要件定義
本シミュレーターが市場で優位性を持つための機能要件：
「評価額」の推定ロジックの実装: ユーザーが評価額を知らない場合、「売買価格」から自動で補正係数（土地0.7、建物0.6など）を掛けて計算する「概算モード」を搭載する。
法務局認定価格テーブルの内蔵: 新築の場合、入力された都道府県と構造から、自動で課税標準を算出する機能（他社ツールにはほぼ存在しない）。
司法書士報酬との合算表示: 税金だけでなく、「登記にかかる諸費用トータル」を見せることで、資金計画ツールとしての価値を高める。
リアルタイム特例判定: 「長期優良住宅」のチェックを入れた瞬間に税額が下がるインタラクティブなUIにより、特例適用のメリットを可視化する。
第10章 ユーザーサポート用FAQデータベース
シミュレーター下部に設置すべき、SEOおよびユーザー補助用のQ&Aコンテンツ案である。
Q1. 登録免許税は誰が払うのですか？
A. 法律上は売主と買主が連帯して納めることになっていますが、実際の不動産取引では、所有権移転（売買）や抵当権設定にかかる費用は**「買主」が全額負担**するのが慣例です。一方、売主側の住所変更や、売主のローン完済に伴う抵当権抹消登記の費用は売主が負担します 2。
Q2. 投資用マンション（賃貸用）でも軽減税率は使えますか？
A. いいえ、使えません。登録免許税の軽減措置（住宅用家屋の特例）は、**「個人の自己居住用（マイホーム）」**であることが絶対条件です。セカンドハウスの場合は、毎月1日以上居住していることを証明する等の厳しい条件をクリアし、自治体の証明書が得られれば適用される場合があります 2。
Q3. 「固定資産税評価額」はどこでわかりますか？
A. 中古物件や土地の場合、所有者（売主）の手元にある「固定資産税納税通知書」の「課税明細書」に記載されています。購入検討中の方は、不動産会社の担当者に「物件の評価額（評価証明書の額）を教えてほしい」と尋ねると確認できます。まだわからない場合は、シミュレーターの「概算モード」をご利用ください 1。
Q4. 軽減措置を受けるために必要な書類は？
A. 最も重要なのは、市区町村役場で発行してもらう**「住宅用家屋証明書」**です。これを取得するために、住民票、売買契約書、登記事項証明書、長期優良住宅の場合はその認定通知書などが必要になります。通常は司法書士が取得代行を行います 11。
Q5. 2026年以降に購入する場合、税金は高くなりますか？
A. 土地の売買にかかる税率は本則2.0%ですが、特例により1.5%に軽減されています。この特例は2026年（令和8年）3月末が期限ですが、政府はこれを2029年（令和11年）まで延長する方針を示しています。住宅の軽減税率（0.3%等）も2027年（令和9年）3月末まで適用されるため、当面は急激な増税はない見込みです 11。
第11章 結論と実装ロードマップ
11.1 結論
登録免許税シミュレーターの開発は、単なる「掛け算ツール」ではなく、法務局の認定価格ロジックや複雑な軽減税率判定ロジックを内包した「エキスパートシステム」として設計すべきである。特に、**「評価額の壁（ユーザーが数値を知らない問題）」**を解決する推定機能の実装が、競合に対する最大の差別化要因となる。
