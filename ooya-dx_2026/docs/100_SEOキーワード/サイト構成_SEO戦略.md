## 推奨サイト構成

### 原則：1ページ = 1キーワード（カニバリ防止）

各ページのターゲットKWが重複しないように設計する。

### 主要ページの役割分担（最終確定）

```
/                  ブランド検索の受け皿。人気ツール6〜8個のみ表示。
                   全ツール一覧は置かない（/tools と重複するため）。
                   「全ツールを見る →」で /tools へ誘導。
/tools             「不動産投資 計算ツール」のハブページ。
                   カテゴリ別全27ツール + 将来アコーディオン記事リンク。
/tools/simulator   「不動産投資 シミュレーション」で順位を取るページ。
                   最重要コンバージョンページ（会員登録の入口）。
```

**分離の理由：**
- /tools のカテゴリ分け（収益性・評価額・ローン・税金・売却・リノベ）は探索性に重要
- 将来アコーディオン記事リンク（27ツール×3〜5記事）をトップに置くと重すぎる
- トップページはブランド＆CTA特化で簡潔に保つ

### カニバリチェック

| | `/` | `/tools` | `/tools/simulator` |
|---|---|---|---|
| 不動産投資 | ○ 本文に含む | ◎ title/h1 | ○ title/h1 |
| 計算ツール | × 使わない | ◎ title/h1 | × 使わない |
| シミュレーション | **× 使わない** | × 使わない | **◎ title/h1** |
| 賃貸経営 | ○ ブランド文脈 | × | △ 本文で補足 |
| 大家DX | ◎ title/h1 | ○ title末尾 | ○ title末尾 |

**重要：トップページのtitleに「シミュレーション」を含めない。**
これを含めると `/tools/simulator` とカニバる。

## 推奨タイトル・メタデータ

### `/`（トップページ）【確定】

```
role:   ブランド検索の受け皿 + EEAT（現役大家の信頼性）
        サイトのコンセプトを一言で伝える
target: 「大家DX」（ブランド名）

title:  不動産投資クラウドソフトなら「大家DX」｜現役大家が開発
h1:     現役大家が作った、不動産投資クラウドソフト
sub:    7物件を購入した現役大家が開発。計算ツールは全て無料で利用可能。
desc:   7物件を購入した現役大家が開発した不動産投資クラウドソフト。
        利回り・税金・ローン・収支分析など27種類の計算ツールを無料で提供。
        スマホでもPCでも、いつでも最新データで計算可能。
```

設計意図：
- 「不動産投資クラウドソフト」= 大家DXが定義する新カテゴリ
  - freee「クラウド会計ソフト」と同じ構造
  - 競合が誰も使っていない → このカテゴリ = 大家DX になる
  - 「クラウド」でExcel・インストール型ソフトとの差別化
  - 「いつでもどこでも最新データで計算」という利便性が伝わる
- titleは「なら大家DX」パターンでブランドポジショニング
- h1はEEAT（現役大家が作った）でGoogle・ユーザー双方に信頼性を伝える
- 「シミュレーション」を入れない（→ /tools/simulator とカニバリ防止）
- 「計算ツール一覧」を入れない（→ /tools とカニバリ防止）
- 「全て無料」はtitleに入れない（将来の有料プラン導入に対応）
- 「無料」はsub/descで補足（titleを変更せずに済む）
- 人気ツール6〜8個のみ表示。全ツール一覧は /tools に誘導。

カテゴリ名の選定経緯：
- 「サイト」→ 抽象的すぎる（楽待のような物件サイトと混同）
- 「ツール」→ /tools の「計算ツール一覧」と微妙に被る
- 「ソフト」→ DL感がある（freeeの「クラウド会計ソフト」で払拭可能）
- 「クラウドソフト」→ ◎ 差別化 + モダン感 + freee型カテゴリ創造

```
role:   全ツールのハブ。ツールを探しているユーザー向け。
target: 「不動産投資 計算ツール」

- 現状のタイトルがKWに最適化されているので変更不要
- 今後アコーディオンで記事リンクを追加する際もtitleは変えない

### `/tools/simulator`（シミュレーター）

```
role:   最重要コンバージョンページ。会員登録の入口。
target: 「不動産投資 シミュレーション」（SV 880, KD 31）

title:  不動産投資シミュレーション【無料】収支・利回り・IRR一括計算｜大家DX
h1:     不動産投資シミュレーション
desc:   不動産投資の収支を無料でシミュレーション。
        IRR・CCR・DSCR・キャッシュフローを35年分一括計算。
        物件価格・家賃・ローン条件を入力するだけで投資判断に必要な指標を即算出。
```

変更前 → 変更後：
- title: 「賃貸経営シミュレーター」→「不動産投資シミュレーション」
- h1: 「賃貸経営シミュレーター」→「不動産投資シミュレーション」
- SV: 20 → **880（44倍のポテンシャル）**

本文中で「賃貸経営の収支計算にも対応」と記載すれば、
「賃貸経営」の文脈も失わない。

## サイト階層と内部リンク構造

```
/ (トップ)
├── /tools (ツール一覧ハブ)
│   ├── /tools/simulator (シミュレーター) ← 会員登録の入口
│   ├── /tools/brokerage (仲介手数料ピラー)
│   │   ├── /tools/brokerage/about
│   │   ├── /tools/brokerage/market-rate
│   │   ├── /tools/brokerage/free
│   │   ├── ...（15記事）
│   │   └── /tools/brokerage/documents
│   ├── /tools/yield-calculator (利回り計算)
│   │   └── （将来：記事群）
│   ├── /tools/loan-calculator (ローン計算)
│   │   └── （将来：記事群）
│   └── ...（27ツール）
├── /media (ブログ記事)
└── /company (会社情報)
```

### 内部リンクの流れ

```
記事 → ピラーページ（計算ツール） → シミュレーター → 会員登録
```

1. 記事（例: /tools/brokerage/about）で「仲介手数料込みの総収支を計算しませんか？」→ シミュレーターへ誘導
2. 計算ツール（例: /tools/brokerage）で「より詳しい収支分析はシミュレーターで」→ シミュレーターへ誘導
3. シミュレーターで計算実行 →「結果を保存するには無料会員登録」→ コンバージョン

### トップページからのリンク構造

```
トップページ
  ├── [CTA] → /tools/simulator（メイン導線）
  ├── [人気ツール] → 上位6〜8ツール（/tools への全件リンクあり）
  └── [最新記事] → /media + ツール記事
```

トップページには全ツールを並べない（/tools と役割が被る）。
人気ツールだけ表示し、「全ツールを見る →」で /tools へ誘導。

```
Phase 1: 認知（SEOトラフィック獲得）
  Google → 記事（15本 × 各ツール） → サイトに来る

Phase 2: 体験（個別ツールで価値を感じる）
  記事 → 計算ツール使用 → 「便利だ」

Phase 3: 深い体験（シミュレーター）
  計算ツール → シミュレーター → 「もっと使いたい」

Phase 4: コンバージョン（会員登録）
  「保存したい」「比較したい」→ 会員登録

| 無料（登録なし） | 会員限定（検討中） |

| 基本計算（IRR, CCR, DSCR） | 計算結果の保存 |
| 1物件のシミュレーション | 複数物件の比較 |
| 標準パラメータ | カスタムパラメータ |
| - | PDFレポート出力 |
| - | 過去シミュレーション履歴 |
