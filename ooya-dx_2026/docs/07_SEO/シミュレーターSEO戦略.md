### 内部リンク構造（トピッククラスター）

```
        /tools/（ピラー）
            │
            ↓ 一覧リンク
    /tools/brokerage/（ツール）
            │
            ↕ 相互リンク
    /media/.../brokerage-guide/（記事）
            │
            ↑ 一覧リンク
        /media/（ピラー）
```

**ポイント：**
- ツールとメディアは分離して管理
- 同じトピックのツールと記事は相互リンク
- 各ピラーページから個別ページへリンク

## ページ設計（各ツール共通）

### 必須要素

| 要素 | 説明 | 例 |
|------|------|-----|
| タイトルタグ | キーワード + ベネフィット | 「仲介手数料計算｜無料シミュレーター｜大家DX」 |
| メタディスクリプション | ユーザー向け説明（120字） | 「不動産売買の仲介手数料を瞬時に計算。消費税込みの金額がすぐわかります。」 |
| H1 | メインキーワード含む | 「仲介手数料シミュレーター」 |
| H2 | サブセクション | 「計算方法」「よくある質問」「関連ツール」 |
| 構造化データ | Schema.org | HowTo, FAQPage, WebApplication |

### コンテンツ構成（推奨5,000字以上）

```
1. シミュレーター本体
   - 入力フォーム
   - 計算結果表示
   - 計算式の説明

2. 解説セクション
   - 〇〇とは
   - 計算方法の詳細
   - 具体例（金額別）
   - 注意点・よくある間違い

3. FAQセクション
   - Q: 消費税は含まれる？
   - Q: 値引きは可能？
   - Q: いつ支払う？
   - など5〜10問

4. 関連情報
   - 関連ツールへのリンク
   - 関連記事へのリンク
   - 最新の法改正情報

5. フッター情報
   - 最終更新日
   - 免責事項
   - 監修者情報（あれば）
```

## 上位表示のための7つの戦略

### 1. コンテンツ量を増やす
- 競合の2倍（5,000字以上）を目標
- 計算式の詳細解説
- 具体例（複数の金額パターン）

### 2. FAQセクション追加
- 5〜10問のQ&A
- 構造化データでリッチスニペット表示

### 3. 構造化データ実装
- WebApplication
- FAQPage
- HowTo

### 4. 内部リンク強化
- ツール間の相互リンク
- メディア記事との相互リンク
- パンくずリスト

### 5. 最新情報の強調
- 法改正情報を目立つ位置に
- 「最終更新日」を表示
- 定期的な更新

### 6. ユーザー体験向上
- 計算結果のコピー/共有機能
- PDF出力機能
- 計算履歴（ログインユーザー）

### 7. Core Web Vitals最適化
- LCP: 2.5秒以内
- FID: 100ms以内
- CLS: 0.1以内

## 細分化の判断基準

派生ページを作るかどうかは、以下で判断：

| 条件 | 判断 |
|------|------|
| 派生キーワードの検索数 > 1,000/月 | 別ページを検討 |
| 派生キーワードの検索数 < 1,000/月 | メインページ内で対応 |
| 派生ページの流入がメインの5%未満 | 不要 |

**例：仲介手数料**
- メイン（速算式）: 35,381 → ✅ 必須
- レーマン方式: 154 → ❌ 不要（メイン内で対応）
- 円単位計算: 80 → ❌ 不要（トグルで対応）
