🎯 実装の特徴

  • エラーフィールドには border-red-300 と bg-red-50 クラスが適用される
  • 通常フィールドには border-gray-300 クラスが適用される
  • 各フィールドには data-field 属性が設定されている
  • エラーメッセージは ErrorMessage コンポーネントで統一表示
  • handleFieldChange 関数で入力時のエラークリアを実現
  • getFieldClassName ヘルパー関数で動的なクラス名生成

📸 実装コードの抜粋

// エラースタイルの動的適用
const getFieldClassName = (fieldName: string, baseClass: string = "...") => {
  return `${baseClass} ${
    fieldErrors[fieldName] 
      ? 'border-red-300 bg-red-50 focus:ring-red-500' 
      : 'border-gray-300 focus:ring-indigo-500'
  }`;
};

// 入力時のエラークリア
const handleFieldChange = (fieldName: string, value: any) => {
  setFieldErrors(prev => {
    const newErrors = { ...prev };
    delete newErrors[fieldName];
    return newErrors;
  });
  handleInputChange(fieldName, value);
};
